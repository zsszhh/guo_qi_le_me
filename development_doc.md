# 食物与药物物品管理应用开发文档

---

## 应用说明

### 这是一款什么应用？

**过期了么**是一款轻量级的物品管理应用，专注于解决日常生活中食品和药品的过期问题。

### 它能做什么？

| 核心功能 | 说明 |
|----------|------|
| 📸 AI智能录入 | 拍照即可自动识别物品信息，彻底告别手动录入的繁琐 |
| ⏰ 保质期追踪 | 自动计算物品保质期，清晰展示物品状态 |
| 🔔 多元提醒 | 过期前3/7/14天自动提醒，再也不会错过最佳使用时间 |
| 🔄 数据同步 | 支持WebDAV备份，多设备数据同步，数据永不丢失 |
| 🤖 智能学习 | AI学习你的使用习惯，越用越懂你 |

### 解决什么痛点？

| 痛点 | 解决方案 |
|------|----------|
| 东西太多记不住 | AI拍照自动识别，一键录入 |
| 不知道什么时候过期 | 自动计算保质期，状态一目了然 |
| 经常错过最佳使用时间 | 多元化提醒，再也不会忘记 |
| 换手机数据丢失 | WebDAV云端备份，换机无忧 |

### 技术特点

- **纯前端架构**：无需后端服务器，部署简单
- **国内AI模型**：支持豆包、通义千问等国内大模型
- **本地优先**：SQLite本地存储，离线可用
- **用户可控**：API Key自配，数据完全由用户掌控

### 适合人群

- 经常购买食品、药品，需要管理保质期的个人用户
- 希望用AI简化物品录入流程的用户
- 注重数据隐私，希望自主掌控数据的用户

## 一、核心功能需求

### 1.1 物品管理基础功能

**物品分类体系**：

- **食品类**：生鲜食品、包装食品、调味品、饮料等
- **药品类**：处方药、非处方药、保健品、医疗器械等

**基础操作**：

- 添加物品（AI智能录入首选，手动录入备选）
- 编辑物品信息
- 删除物品（支持批量删除）
- 物品详情查看

### 1.2 AI智能录入功能（核心首选方式）

**设计理念**：以多模态大模型为核心，通过智能交叉核验agent实现精准、便捷的物品信息录入，彻底解决录入麻烦的核心痛点。

**多模态智能录入（首选方式）**：

- 一键拍摄物品包装照片
- 多模态大模型自动识别图像中的物品名称、品牌、规格
- OCR提取生产日期/保质期信息
- 智能判断物品类别（食品/药品）并自动分类
- 自动计算过期日期
- 用户仅需确认/微调即可保存

**语音录入（第二选择）**：

- 自然语言语音输入（如："牛奶，5月1日购买，保质期6个月"）
- 多模态大模型语音转文字并解析关键信息
- 自动识别日期格式、物品名称和数量
- 智能填充所有字段

**智能交叉核验Agent**：

- 图像识别结果与语音解析结果交叉验证
- 结合历史数据进行一致性检查
- 自动修正识别误差
- 对不确定的信息智能询问用户确认

**历史记录学习**：

- 根据历史录入记录智能推荐常用物品
- AI学习用户的物品偏好和购买习惯
- 下次录入时自动填充高频物品信息
- 保质期智能预测（根据物品类型和历史数据）

**智能补全**：

- 输入物品名称时自动补全
- 根据品牌历史记录自动填充规格
- 智能推荐存放位置

### 1.3 智能化辅助功能

**智能提醒优化**：

- AI分析用户习惯，自动调整最佳提醒时间
- 根据物品重要性和用户历史处理行为智能排序提醒
- 重复购买提醒（基于物品消费周期预测）
- 智能合并同类物品提醒，减少打扰

**智能分类管理**：

- AI自动识别物品类别和子类别
- 智能推荐存放位置（根据物品类型和使用频率）
- 自动识别相近物品并提示合并

**健康洞察与分析**：

- 食品过期原因分析（提醒是否及时处理）
- 消费习惯报告（高频物品、浪费统计）
- 药品使用追踪（服用周期提醒）
- 营养摄入建议（基于食品分类统计）

**智能搜索与筛选**：

- 自然语言搜索（如："上周买的牛奶"）
- 智能推荐相关物品
- 图片搜索（拍照即可找到相似物品记录）

### 1.4 保质期追踪功能

**时间节点记录**：

- 购买日期
- 开封日期（可选）
- 过期日期
- 保质期时长

**状态分类**：

| 状态      | 含义   | 判断逻辑                       |
| ------- | ---- | -------------------------- |
| ✅ 正常    | 状态良好 | 过期日期 - 当前日期 > 30天          |
| ⚠️ 即将过期 | 需关注  | 过期日期 - 当前日期 ≤ 30天 && > 14天 |
| 🔴 急需处理 | 尽快使用 | 过期日期 - 当前日期 ≤ 14天 && > 0天  |
| ❌ 已过期   | 不可使用 | 过期日期 - 当前日期 ≤ 0天           |

**AI智能状态预测**：

- 预测物品实际新鲜度（考虑存储条件）
- 开封后保质期自动缩短计算
- 智能判断是否需要加速使用

**统计分析**：

- 物品过期率统计
- 各类物品库存统计
- 消费习惯分析
- AI智能推荐购买清单

### 1.5 多元化提醒机制

**智能提醒时间**：

- 过期前3天（紧急提醒）
- 过期前7天（重要提醒）
- 过期前14天（常规提醒）
- AI智能推荐最佳提醒时间（根据用户历史行为）

**提醒方式**：

- 应用内通知（弹窗+消息列表）
- 系统推送通知（iOS/Android）
- 桌面小组件提醒
- 智能铃声分类（区分食品/药品提醒）

**AI智能提醒策略**：

- 按物品类型自动设置不同提醒策略（药品更频繁）
- 临近过期物品自动提升提醒优先级
- 重复提醒智能间隔（避免过度打扰）
- 基于位置提醒（到家时提醒处理易过期物品）

**提醒设置**：

- 支持按物品类型设置不同提醒策略
- 可自定义提醒时间
- 支持提醒开关控制
- 免打扰时段设置

### 1.6 数据存储与备份

**本地优先存储**：
- SQLite本地数据库（主要数据存储）
- 离线使用完全支持
- 快速响应，无网络依赖

**WebDAV数据同步（当前阶段）**：
- 用户自行配置WebDAV服务器地址
- 自动定期同步数据库文件到WebDAV
- 支持手动触发同步
- 数据加密传输（HTTPS）
- 跨设备数据同步（多设备共享同一WebDAV）
- 支持主流WebDAV服务（Nextcloud、OwnCloud、坚果云等）
- 数据完全由用户自己控制，隐私性高

**云端同步服务（可选扩展）**：
- 支持配置自定义云端服务器地址
- 自动云端备份功能
- 多设备实时同步
- 数据加密传输和存储
- 可选择开启或关闭

**数据备份（本地）**：
- 自动定期备份数据库文件到本地存储
- 支持手动导出数据（JSON格式）
- 支持数据导入功能
- 换机迁移便捷

**AI服务配置**：
- 用户自行配置大模型API Key（豆包/通义千问/智谱等）
- 支持多模型选择
- API Key本地加密存储
- 数据完全由用户掌控

**优势**：
- 纯前端应用，无需后端服务器
- WebDAV同步：数据完全由用户自己控制
- 云端同步：支持自定义服务器，灵活选择
- 轻量级，部署简单
- 跨平台支持（iOS/Android/桌面）

### 2.3 视觉设计规范

| 紧急 | Critical Red | `#b91a24` | 紧急/已过期状态 |
| 背景 | Surface | `#f9f9ff` | 页面背景 |
| 文字 | On Surface | `#161c27` | 主要文字 |

| 急需处理 | Critical | `#b91a24` | 🔴 |
| 已过期 | Error | `#ba1a1a` | ❌ |

```dart
// AI服务调用示例
class AIService {
  final AIConfig _config;

  Future<ItemInfo> recognizeImage(Uint8List imageData) async {
    // 1. 从加密存储获取API Key
    final apiKey = await _getStoredApiKey();

    // 2. 构建请求（以豆包Seed为例）
    final response = await dio.post(
      'https://ark.cn-beijing.volces.com/api/v3/responses',
      options: Options(headers: {
        'Authorization': 'Bearer $apiKey',
        'Content-Type': 'application/json',
      }),
      data: {
        'model': 'doubao-seed-2-0-lite-260215',
        'input': [
          {
            'role': 'user',
            'content': [
              {
                'type': 'input_image',
                'image_url': 'data:image/jpeg;base64,${base64Encode(imageData)}'
              },
              {
                'type': 'input_text',
                'text': '识别图片中的物品信息，返回物品名称、品牌、规格、生产日期、保质期'
              }
            ]
          }
        ]
      },
      queryParameters: {'timeout': _config.timeoutSeconds},
    );

    return _parseResponse(response);
  }
}
```

1. **数据采集**：前端调用摄像头/麦克风采集图像和语音
2. **多模态解析**：直接调用大模型API进行统一解析（用户配置API Key）
3. **交叉验证**：比对不同输入源的结果一致性
4. **历史比对**：与本地SQLite历史记录进行相似度匹配
5. **误差修正**：自动修正识别误差
6. **智能确认**：对置信度低的信息询问用户确认

**技术实现**：

- **前端**：Flutter调用设备摄像头/麦克风采集数据
- **Agent逻辑**：Dart实现交叉核验算法
- **大模型调用**：直接调用豆包Seed/通义千问VL等国内API
- **数据存储**：SQLite存储历史数据供比对

### 4.3 智能交叉核验Agent实现（Dart）

```dart
// Agent核心逻辑示例
class SmartVerificationAgent {
  final DatabaseService _dbService;
  final AIService _aiService;

  SmartVerificationAgent(this._dbService, this._aiService);

  Future<ItemInfo> verifyItem(MultiModalInput input) async {
    // 1. 调用多模态大模型解析
    final parsedResult = await _aiService.callMultiModalModel(input);

    // 2. 历史数据模糊搜索
    final similarItems = await _dbService.fuzzySearch(parsedResult.name);

    // 3. 交叉验证
    final verifiedResult = _crossValidate(parsedResult, similarItems);

    // 4. 智能确认（如有不确定信息）
    if (!verifiedResult.confident) {
      return await _askUserConfirmation(verifiedResult);
    }

    return verifiedResult;
  }
}
```

### 7.1 部署架构（纯前端方案）

```
┌─────────────────────────────────────────────────────────────┐
│                        客户端（Flutter）                      │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐ │

│  │数据库       │  │             │  │  自定义服务器        │ │
│  └─────────────┘  └─────────────┘  └─────────────────────┘ │
│         │                │                    │              │
└─────────┼────────────────┼────────────────────┼──────────────┘
          │                │                    │
          ▼                ▼                    ▼
   ┌────────────┐   ┌────────────┐      ┌────────────┐
   │ 本地存储    │   │ WebDAV服务器│      │ 云端服务器  │
   │            │   │Nextcloud等 │      │（用户自建） │
   └────────────┘   └────────────┘      └────────────┘
                                                    │

                                           ┌────────────┐
```

**可选扩展**：

### 7.3 数据备份方案

**WebDAV同步（当前阶段）**：
- 定期自动同步数据库文件到用户配置的WebDAV服务器
- 支持手动触发同步
- 数据加密传输（HTTPS）
- 跨设备数据同步（多设备共享同一WebDAV）
- 支持主流WebDAV服务（Nextcloud、OwnCloud、坚果云等）
- 数据完全由用户自己控制

**WebDAV同步策略**：
- 同步时机：应用启动时、进入首页时、进入设置页时
- 同步方式：全量同步（当前阶段）、增量同步（后续优化）
- 后台同步：支持后台自动同步（根据设置的间隔）

**WebDAV同步冲突处理策略**：
| 冲突场景 | 处理策略 | 说明 |
|----------|----------|------|
| 本地修改 vs 远程未修改 | 直接上传 | 上传本地最新数据到远程 |
| 远程修改 vs 本地未修改 | 直接下载 | 下载远程最新数据到本地 |
| 双方都修改 | 最后修改优先 | 以时间戳最新为准 |
| 同步失败 | 本地缓存 | 保留本地修改，下次同步重试 |

**云端同步（可选扩展）**：
- 自动备份到用户配置的自定义云端服务器
- 多设备实时同步
- 数据加密传输和存储
- 可选择开启或关闭

**本地备份**：
- 定期自动备份数据库文件到设备存储
- 支持手动导出数据（JSON格式）
- 支持数据导入功能

**优势**：
- WebDAV同步：数据完全由用户自己控制
- 云端同步：支持自定义服务器，灵活选择
- 本地备份：换机迁移便捷
- 多种备份方式，满足不同需求

### 7.4 安全配置

- API Key使用flutter_secure_storage加密存储
- 本地SQLite数据库文件加密
- 数据传输采用HTTPS
- WebDAV认证信息加密存储
- 云端同步认证信息加密存储
- 云端服务器连接支持自签名证书（可选配置）
